# Diagnóstico de Automação e Estimativa de Software

Documentação central do projeto.

## Estrutura

- `phase-a-specification.md`: especificação funcional e arquitetural.
- `phase-b-implementation-base.md`: escopo e decisões da fase B.
- `roadmap.md`: backlog por fase.

## Fases

1. Fase A: especificação e arquitetura.
2. Fase B: bootstrap técnico e módulos base.
3. Fase C: MVP funcional.
4. Fase D: refinamento de UX, templates e robustez.

## Observação

O ambiente atual não possui `node`/`npm` disponíveis no terminal. Por isso, a base foi estruturada manualmente para ficar pronta para instalação das dependências e execução assim que o runtime estiver disponível.
